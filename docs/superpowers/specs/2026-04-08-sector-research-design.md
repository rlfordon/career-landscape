# Sector Research — Deep Research Reports for Career Landscape Profiles

## Problem

The current sector profile descriptions in `js/data.js` are informed by a single sector analysis document but haven't been verified against real-world evidence. Some claims don't hold up under scrutiny (e.g., mass tort junior "case development" is actually intake screening; self-service risk assertions may miss emerging threats). Descriptions lack specificity and source attribution.

## Goals

1. Produce a research document per sector that verifies or challenges current ratings and descriptions against real-world evidence.
2. Update `data.js` descriptions to reflect research findings — more specific, more grounded.
3. Add a `references` array to each sector profile with 3-5 sources for users to explore further.

## Approach: Pilot Then Scale

Run 3 pilot sectors to validate the research process before applying it to all 28 profiles.

**Pilot sectors:**
- Insurance defense — lots of public commentary, should be easy to find evidence
- Plaintiffs' mass tort — already partially investigated, tests whether the process catches known issues
- Public defender / legal aid — unique political economics, tests whether the template works for non-commercial sectors

**Execution order:**
1. Insurance defense end-to-end as proof of concept
2. If process works, mass tort and public defender in parallel
3. Review all three, decide whether to scale to remaining 25 sectors

## Research Document Structure

Each sector gets a file at `docs/research/{sector-id}.md`:

```
# {Sector Name} — AI Impact Research

## Current Profile Summary
[Snapshot of current ratings and descriptions]

## Standard Research Findings
### What do junior/senior attorneys actually do?
### What AI tools exist for this sector?
### What are clients/counterparties doing with AI?
### Industry commentary on AI impact

## Sector-Specific Findings
### [Skeptical question 1]
### [Skeptical question 2-3]

## Recommended Changes
[Specific rating or description changes with justification]

## References
[Numbered list of sources with URLs and one-line descriptions]
```

## Research Questions Per Sector

**Standard questions (every sector):**
1. What are junior vs senior attorneys actually doing day-to-day in this sector?
2. What AI tools exist or are being built for this sector specifically?
3. What are clients/counterparties doing with AI that could displace or pressure lawyers in this sector?
4. What does industry commentary (legal press, law professors, consultants) say about AI's impact on this sector?

**Sector-specific questions:** Generated by reading the current profile descriptions and identifying claims that are asserted without evidence, seem too optimistic or pessimistic, or could be challenged by plausible counter-scenarios.

## Data Model Changes

Add a `references` array at the sector level (not per junior/senior):

```javascript
'insurance-defense': {
    id: 'insurance-defense',
    name: 'Insurance Defense',
    references: [
      { title: 'Short descriptive title', url: 'https://...', source: 'Bloomberg Law' },
      { title: 'Another article', url: 'https://...', source: 'Law360' },
    ],
    junior: { ... },
    senior: { ... },
}
```

3-5 references per sector targeting accessible, informative articles about how AI is affecting the sector.

## Workflow Per Sector

1. **Draft questions** — Read current profile, draft 2-3 skeptical sector-specific questions
2. **Run research agent** — One agent with all standard + specific questions, web search enabled
3. **Write research doc** — Compile findings at `docs/research/{sector-id}.md`
4. **Review together** — Present summary of what research supports, contradicts, and recommended changes
5. **Update data.js** — Revise descriptions, adjust ratings if warranted, add references array

## Out of Scope

- UI changes to display references (separate task after pilots)
- Changes to the O-Ring framework or dimension definitions
- Research on sectors beyond the 3 pilots (until process is validated)
- Polished prose in research docs — they're working notes
